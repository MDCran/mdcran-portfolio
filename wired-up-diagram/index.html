Device Fingerprint Engine

lib/device-fingerprint.ts
🖼
canvas Text rendered to a hidden canvas, hashed — unique per GPU/driver/OS font rendering pipeline
🎮
webgl-vendor · webgl-renderer GPU make/model from WebGL debug extension (e.g. "NVIDIA GeForce RTX 4080")
🖥
screen Width × height, device pixel ratio, color depth — e.g. "1920x1080@2"
🕐
timezone Intl.DateTimeFormat().resolvedOptions().timeZone (e.g. "America/New_York")
🌐
language · languages[] navigator.language (primary) + full navigator.languages preference array
⚙️
hardwareConcurrency Logical CPU core count (navigator.hardwareConcurrency)
💾
deviceMemory Approximate RAM in GB (navigator.deviceMemory) — rounded to nearest bucket
🔍
userAgent · platform Full UA string + navigator.platform — parsed into browser/OS/device type
OUTPUT: All signals joined → SHA-256 → 32-char hex serial  ·  Stable across reloads  ·  Survives cookie + cache clear  ·  Falls back to FNV-1a on non-HTTPS  ·  Prefers-color-scheme deliberately excluded (would break on theme toggle)

Cross-Device Linking Methods

lib/identity.ts · lib/cross-device.ts · lib/handshake.ts
Method Trigger Confidence linkMethod value Notes
Tracking Link ?identity=ID in URL (shared by owner)
100% — deterministic
"link" URL param captured by IdentityTracker, stripped after claim. Fires mdcran:identity-claimed event.
Same Device Serial Returning browser, identical fingerprint
100% — same browser
"confirmed" Primary recognition method. Stable even after cache clear.
QR Handshake Bridge __SCANTOMOBILE__ marker → QR → phone scans → token claim
100% — token exchange
"handshake" Desktop generates token in handshakes collection; mobile claims it within TTL, merges devices[].
Same-IP Suggestion Multiple named identities detected on same network IP
50–85% — probabilistic
"ip-suggested" suggestionsForIp() — excludes auto-created anonymous identities. User must confirm. Denials tracked in identityDenials.
AI Name Extraction AI infers name from conversation, user confirms
Medium-High (post-confirm)
"ai" AI uses __IDENTITY_PROVIDE:name__. Origin stored as "ai" on claim. Discord notified with aiContext.
Admin Manual Merge Dashboard → Identities → move-device action
100% — manual
"admin" moveDeviceToIdentity() in identity lib. Discord notified with origin "admin".
Cross-device probabilistic scoring (evaluateDeviceLinks() in lib/cross-device.ts): shared browsed paths, hardware match (GPU, screen), timing correlation, language match → score 50–85% triggers soft probe in AI chat via identityContext.linkCandidate. AI presents bridge as __SCANTOMOBILE__ or asks user to confirm.

AI Chat — Context Variables Injected per Request

app/api/chat/route.ts
visitorName string | null Confirmed name from identity system. AI greets visitor by name.
visitorGeo string City/region derived from IP geolocation (e.g. "Miami, FL")
language string User-selected language code. AI responds in this language.
tone "friendly" | "professional" | "concise" Conversation tone set via accessibility panel or chat.
remember.visits number Total session count for this visitor (from localStorage)
remember.returning boolean Whether this is a return visit — AI acknowledges returning visitors.
remember.daysSinceLast number | null Days since previous visit — AI can reference recency contextually.
remember.daypart "morning" | "day" | "evening" | "night" Time-of-day for contextual greeting ("Good evening…")
remember.topics string[] Last 4 conversation topics from prior sessions (localStorage). AI references prior conversations.
domContext string Extracted DOM structure of the current page — sections, projects, content visible to user.
currentPage string URL path the user is currently viewing (e.g. "/projects")
agentName string Persona name — default "Michael". First-person voice of the site owner.
referrerContext.resolvedSource string Traffic source category (LinkedIn, Google, GitHub, Direct, etc.)
referrerContext.utmSource / utmCampaign / utmMedium string | null UTM parameters from the URL — campaign-aware greetings.
identityContext.state "confirmed" | "suggested" | "anonymous" Current identity recognition level. Drives how AI addresses visitor.
identityContext.confirmedName string | null Real name if state is "confirmed".
identityContext.suggestedName string | null Candidate name from IP suggestion (50-85% confidence).
identityContext.suggestedCertainty number (0–1) Confidence score for the suggestion.
identityContext.linkCandidate object | null Cross-device bridge hint: { serial, score, sharedPath, criteria }. AI can offer __SCANTOMOBILE__.
sessionId · deviceSerial string Session UUID and device fingerprint serial — for rate-limiting and identity correlation.
AI Provider Chain:   1. Anthropic Claude (primary, streaming) → 2. OpenRouter fallback → 3. OpenAI fallback  ·  Auto-switches on rate-limit / error  ·  Configured in lib/db.ts → chatConfig  ·  Model IDs stored in DB, admin-configurable

AI Markers — Full Command Reference

app/api/chat/route.ts · ChatPanel.tsx
The AI embeds __MARKER__ tokens in its response text. ChatPanel.tsx intercepts and executes them client-side. The AI only uses portfolio-relevant actions — never breaks out of the site context.
Navigation & View Control
__NAV:/path__Navigate browser to a page path — no announcement shown to user
__HIGHLIGHT:target__Scroll to + spotlight element with animated cursor and pulse ring
__ZOOM:target__Zoom/focus on a specific element (scale transform + backdrop)
__RESETZOOM__Reset any active zoom back to normal view
__EMPHASIZE:target__Subtly emphasize an element (lighter highlight than zoom)
__SCROLL:direction[:containerId]__Scroll up / down / left / right / top / bottom — optional container
__PREFETCH:/path__Silently preload a page (Next.js router.prefetch) for instant future navigation
Content Interaction & Forms
__IMAGESHOW:N__Open image N in the Lightbox component for full-screen view
__CLICK:element-label__Simulate a click on an element matched by aria-label / text content
__TYPE:field-label|text__Type into a form field matched by label (can chain multiple)
__TYPEID:fieldId|text__Type into a field by ID — most reliable method
__SLIDER:targetId|percent__Set a range slider to 0–100 percent value
__CHECK:targetId|on/off__Toggle a checkbox or radio button on or off
__SELECT:targetId|option__Select a dropdown option by value or visible text
__COPYLINK__ / __COPYLINK:url__Copy current page URL (or specific URL) to clipboard
Projects & Tours
__PROJECTCARD:projectId__Embed an interactive project card inline in the chat response
__TOUR__Start the full guided site tour (AssistantTutorial component)
__PROJECTTOUR__Start the projects-specific tour (ProjectsTour component)
Identity Management
__IDENTITY_PROVIDE:name__User provided their name during conversation — triggers claim flow
__IDENTITY_CONFIRM:name__User confirmed a suggested identity — calls /identity/claim with existing ID
__IDENTITY_DENY__User denied a suggested identity — records in identityDenials, removes suggestion
__REJECT_CANDIDATE:serial__User rejects a cross-device bridge candidate — suppresses that serial from future probes
__SCANTOMOBILE__Show QR code to bridge desktop session to mobile device (handshake flow)
Accessibility & Theming
__THEME:themeid__Switch site theme: hacker / dark / light / cyberpunk
__TEXTSIZE:value__Adjust font size: small / normal / large / larger / largest
__ACCESS:flag__Toggle accessibility: cb-deuteranopia, cb-protanopia, cb-tritanopia, readaloud-on/off, motion-reduce/normal, reset
__CONSENT:opt-in/opt-out__Set cookie consent preference via AI command
Contact & Safety
__CONTACTCARD:{json}__Submit a pre-filled contact form (name, email, message) on user's behalf
__BOOKINGCARD:{json}__Submit a pre-filled booking request (name, email, date, time, notes)
__BEHAVIOR__Safety violation detected — AI must respond with ONLY this marker; chat handles display of safety notice
